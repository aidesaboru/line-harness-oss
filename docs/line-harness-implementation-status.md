# LINEハーネス 実装状況まとめ

更新日: 2026-06-25

## 1. いまの目的

LINEハーネスは、LINE公式アカウントで毎日行っている顧客対応を、あとから追えるようにするための管理画面です。

今回の改善では、特に次の3つを優先しています。

1. どの問い合わせを、誰が、次に対応するのか分かるようにする
2. 二次対応に回した相談が埋もれないようにする
3. LINE公式側だけを見ないと分からない情報を、LINEハーネス側にも残す

## 2. 反映済みのこと

### サポートCRMの見やすさ

サポートCRM上部の分類を、日常運用で使う5つに整理しました。

- 進行中の案件
- 二次対応が確認中
- 一次対応が対応中
- 顧客返信待ち
- 完了

これにより、「今どこで止まっているか」を先に見て、細かい条件はあとから確認する形にしています。

### 見落とし防止

サポートCRMに「見落とし注意」の表示を追加しました。

ここでは、次のような危ない案件をすぐ見つけられます。

- 大至急の案件
- 期限を過ぎている案件
- 4時間以内に期限が来る案件
- 担当者が入っていない案件

### 二次対応の流れ

案件の状態を「誰が次に動く番か」で見られるようにしました。

- 二次対応が確認中: 詳しい人が確認する番
- 一次対応が対応中: 一次対応者が顧客へ返す番
- 顧客返信待ち: 顧客からの返事待ち
- 完了: 対応終了

二次対応者が回答したら、案件を一次対応側へ戻せる形になっています。

### 顧客カード

個別チャットの右側に、顧客カードを追加しました。

入れられる情報は以下です。

- 顧客番号
- 法人名
- 担当者名
- 店舗名
- 引き継ぎ日
- 契約内容
- 最低保証
- 複数店舗メモ

必須情報が足りない顧客は、友だち一覧でも分かるようにしています。

### 顧客情報の一括更新

友だち管理画面から、CSVまたはTSVを貼り付けて、顧客情報をまとめて更新できるようにしました。

1件ずつ手入力しなくても、既存顧客の情報をまとめて入れられます。

一斉送信から外したい顧客は、顧客カードの「一斉送信から除外」で設定できます。CSV/TSV一括更新でも `broadcastExcluded` または `一斉送信除外` の列を使って、まとめて除外・解除できます。

### LINE履歴の見え方

個別チャットと案件詳細で、送信元が分かるようにしました。

例:

- LINE公式
- 一斉送信
- 自動返信
- 自動化
- 手動送信

画像はチャット内でプレビューできます。PDFやファイルは、開く・確認するための表示にしています。

### LINE公式から送った文章の扱い

LINE公式アカウント画面から送った文章は、LINE側の仕様上、LINEハーネスが自動で完全取得できる前提にはできません。

そのため、現時点では「公式送信を記録」ボタンで、重要な返信内容をLINEハーネス側に残す運用にしています。

### 既読について

LINEハーネス内でチャットを見るだけでは、LINE公式側のチャット画面に勝手に既読を付ける操作にはしていません。

個別返信時は、送信時に既読を付けるかどうかを選べる形にしています。

### 左メニューの整理

日常的に使う項目を上に寄せました。

上に置いたもの:

- 未対応
- 個別チャット
- サポートCRM
- 一斉送信
- 自動返信ルール
- 友だち管理

普段あまり使わない機能は、下側に回す方針にしています。

### 緊急時の安全停止

緊急コントロールに「LINE送信セーフティ停止」を追加しました。

ONにすると、対象アカウントからの以下の送信を止めます。

- 個別返信
- 友だち詳細からの送信
- 一斉送信
- 自動返信
- シナリオ配信
- リマインド配信

BANリスクや誤配信が疑われる場合、まずここをONにして止血できます。

### Slack通知

サポートCRMからSlack通知を設定できるようにしました。

できること:

- Slack Webhook URLを保存する
- 大至急の案件だけ即時通知する
- 未完了案件を12時、14時、17時などの指定時刻にまとめて通知する
- 担当者別の未対応件数を通知する
- 期限超過、期限間近、大至急の件数を通知する
- 通知からサポートCRMを開ける

Slack Webhook URLが未設定の間は通知を送りません。URLを入れて有効化すると動きます。

### 一斉送信の安全除外

一斉送信で、送らない方がよい顧客を自動で対象から外すようにしました。

対象から外すもの:

- LINEの友だち状態が停止済みの顧客
- 未完了のサポート案件がある顧客
- 個別チャットが未対応、または対応中の顧客
- 顧客情報に「配信しない」系のフラグが入っている顧客

送信前に表示される対象人数と、実際に送る人数も同じ条件で数えます。

対象になる配信:

- 全員配信
- タグ配信
- 条件で絞った配信
- 複数アカウント重複除外配信

これにより、クレーム対応中の顧客や、配信停止扱いの顧客へ案内を送ってしまうリスクを下げています。

## 6. まだ運用確認が必要なこと

### Slack通知の運用設定

決めること:

- 通知先のSlackチャンネル
- 通知する時刻
- 大至急を誰に即時通知するか
- 通常案件をどこまで通知するか

コード側は実装済みです。実運用では、Slack側でWebhook URLを発行して、サポートCRMの「Slack通知」設定に貼り付けます。

### LINE公式から送った文章

自動取得は前提にしない方が安全です。

運用としては、重要な返信はLINEハーネスから送るか、LINE公式から送った場合は「公式送信を記録」で残す形にします。

### 顧客情報の必須項目

現状は以下を必須扱いにしています。

- 顧客番号
- 法人名
- 担当者名
- 店舗名

実運用で不要な項目があれば、必須から外せます。

### スタッフごとの見える範囲

管理者は全案件を見られます。

スタッフは、自分が作成した案件、自分が担当者に入っている案件、自分が二次対応先になっている案件を見られる設計です。

この判定にはスタッフ名を使うため、スタッフ管理に登録する名前と、案件の担当者名をそろえる必要があります。

## 7. 次にやること

次は、実際の運用メンバーで管理画面を触りながら、次の4点を確認します。

1. サポートCRMの5分類で迷わず案件を追えるか
2. 二次対応者が自分宛の案件だけを見て回答できるか
3. 顧客カードの必須項目が運用に合っているか
4. Slack通知をどの時刻・どのチャンネルに出すか

ここまで確認できれば、LINEハーネスは「実運用前の調整段階」から「チームで使い始める段階」に進められます。
